## 4.1 ארכיטקטורה כללית

המערכת מחולקת ל-6 שכבות עיקריות:

```
שכבה 1 — כניסה לתוכנית
    Program.cs → Application.Run(MainMenuForm)

שכבה 2 — ממשק משתמש (Forms)
    MainMenuForm ← תפריט ראשי מונפש
    mainWin      ← לולאת המשחק הראשית (7 partial classes)
    TrainingForm ← ממשק אימון הבינה המלאכותית

שכבה 3 — ליבת המשחק
    Core/Player        ← פיזיקה, בריאות, מצב super
    Core/GameManager   ← מצב משחק (Running/Paused/Dead/Win)
    Core/GameData      ← הגדרות Mushroom, Coin, QuestionBlock
    Core/TextureLoader ← טעינת sprite-sheets

שכבה 4 — ישויות
    Enemies/*    ← 6 סוגי אויבים
    World/GameObjectS  ← פלטפורמות וצינורות

שכבה 5 — נכסים
    assets/textures/sprite_sheets/*.png

שכבה 6 — בינה מלאכותית
    ML/NetParams
    ML/Neuron, Layer, NeuralNetwork
    ML/MarioAgent
    ML/Population
    ML/NeuralNetworkControl
```

## 4.2 פירוק mainWin ל-7 Partial Classes

`mainWin` היא ה-Form הראשית. כדי להימנע מקובץ ענק, היא מחולקת לקבצים:

| קובץ | אחריות |
|------|--------|
| `mainWin.cs` | שדות, לולאת משחק, קלט מקלדת |
| `mainWin.LevelData.cs` | מערכי שלבים, הגדרות בלוקי שאלה, templates פרוצדורליים |
| `mainWin.LevelBuilder.cs` | בניית שלב, ניקוי שלב, יצירת פלטפורמות |
| `mainWin.Physics.cs` | פתרון התנגשויות, מצלמה, מוות |
| `mainWin.EnemyUpdates.cs` | spawn והעדכון של כל 6 סוגי האויבים |
| `mainWin.Collectibles.cs` | עדכון מטבעות ופטריות |
| `mainWin.HUD.cs` | שרטוט sprite השחקן, overlay ניקוד/חיים |

> כל 7 הקבצים מתקמפלים לכיתה אחת — `partial class` של C#.

## 4.3 תרשים זרימה — לולאת המשחק (Game Loop)

```
gameTimer מפעיל כל 16ms:
    │
    ▼
PhysicsStep()
    ├─ player.Move(direction, jumpEdge, jumpHeld)
    ├─ CheckPlatformCollisions()  [שחקן מול כל פלטפורמה]
    ├─ UpdateGoombas()
    ├─ UpdateKoopas()
    ├─ UpdateFastEnemies()
    ├─ UpdateJumpingEnemies()
    ├─ UpdatePatrolEnemies()
    ├─ UpdateFlyingEnemies()
    ├─ UpdateCoins()
    └─ UpdateMushrooms()
    │
    ▼
UpdateCamera() → bool (האם המצלמה זזה?)
    │
    ├─ אם כן: Invalidate() [ציור מחדש]
    │
    ▼
UpdateHud()   [עדכון תוויות ניקוד/חיים]
```

**אופטימיזציית ביצועים מפתח**: `SuspendLayout()` נקרא פעם אחת בתחילת כל PhysicsStep ו-`ResumeLayout()` בסוף — במקום 7 פעמים בתוך כל תת-פונקציה. חסכון: ~7× קריאות layout לכל tick.

## 4.6 ארכיטקטורת הבינה המלאכותית

### מבנה הרשת

```
קלט I1: dist_gap    ─►┐
קלט I2: dist_enemy  ─►│  שכבה 1     שכבה 2       פלט
קלט I3: height_diff ─►│  [6 נוירונים]─►[4 נוירונים]─►  O1: כיוון
קלט I4: is_grounded ─►┘                             O2: קפיצה

כל חיבור = משקולה (weight)
כל נוירון = tanh(Σ w_i × x_i + bias)
```

### תהליך אבולוציה

```
דור N:
  ┌─ 60 סוכנים רצים עד מוות ─┐
  │                           │
  │  כל סוכן: fitness = max X │
  │                           │
  └─────────────────────────── ┘
                │
                ▼
  מיין לפי fitness
                │
    ┌───────────┴──────────┐
    │  30% עליונים שורדים  │
    └───────────┬──────────┘
                │
    ┌───────────▼──────────┐
    │  70% נוצרים מחדש:    │
    │  crossover + mutation │
    └───────────┬──────────┘
                │
                ▼
            דור N+1
```

## 4.7 תרשים רצף — tick אחד של אימון

```
_simTimer (16ms)
    │
    ▼ לכל סוכן חי:
    ├─ ComputeInputs(world) → [dist_gap, dist_enemy, height_diff, grounded]
    ├─ Brain.Forward(inputs) → [dir_tanh, jump_bool]
    ├─ Agent.Step(dir, jump) [פיזיקה זהה לשחקן]
    └─ ApplyPlatformCollisions(agent)
    │
    ▼ לאחר עדכון כולם:
    ├─ עדכן _cameraX לפי הסוכן המוביל
    ├─ אם כולם מתו → CreateNewGeneration()
    ├─ עדכן Dashboard (דור, חיים, best)
    └─ _canvas.Invalidate()
```

## 4.8 ניהול Z-Stack (סדר ציור)

ב-WinForms, סדר ציור הקונטרולים קובע מה יופיע מעל מה:

```
(עליון — נראה ראשון)
    HUD labels (ניקוד, מטבעות, חיים)
    Player (picboxplayer)
    Enemies (6 סוגים)
    Coins
    Question Blocks
    Platforms
    Bricks / ground strip
(תחתון — רקע)
    Form background
```

> **באג היסטורי שתוקן**: קריאות `SendToBack()` בפונקציות spawn של אויבים גרמו לאויבים להיות מוסתרים מאחורי הרצפה. הפתרון: הסרת ה-`SendToBack` ו-`BringToFront()` לשחקן בלבד.
