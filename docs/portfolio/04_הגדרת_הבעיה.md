# הגדרת הבעיה

---

## 2.1 מהי הבעיה?

פרויקט זה עוסק בשתי בעיות מרכזיות שלכל אחת מהן ייחוד תכנותי:

### בעיה א' — בניית משחק פלטפורמה אותנטי

בניית משחק פלטפורמה בדרגת איכות ראויה ללא שימוש במנוע משחקים מוכן (Unity, Godot וכד'), תוך שמירה על:

- **תנועה אמינה**: תחושה שמרבית גיימרים מזהים כ"נכונה" — תאוצה הדרגתית, עצירה חלקה, קפיצה עם גובה משתנה
- **אויבים חכמים**: כל אויב מתנהג שונה — הולך, קופץ, סייר על פלטפורמה, טס
- **שלבים מעוצבים**: שלושה שלבים שונים בסגנון + שניים פרוצדורליים
- **ביצועים**: 60 פריימים לשנייה ללא תקיעות, גם כאשר מאות אובייקטים על המסך

### בעיה ב' — בינה מלאכותית ללא פקודות ידניות

בהינתן סוכן בעולם הפלטפורמה, **כיצד ניתן ללמד אותו לשחק** — להגיע כמה שיותר ימינה — מבלי לכתוב לו במפורש "קפוץ כשיש פי", "הזז ימינה"?

**האתגר**: הסוכן קולט רק 4 ערכים מספריים (ראו פרק 5), ועליו להחליט בכל 16ms — לאן לנוע, האם לקפץ. מרחב ההחלטות הוא אינסופי, ופונקציית הציון (fitness) פשוטה בכוונה: כמה ימינה הגעת?

## 2.2 למה זה מורכב?

### מורכבות הפיזיקה

פיזיקה פשוטה (מיקום += מהירות) מייצרת תנועה "מרחפת" ולא טבעית. המשחק המקורי של Nintendo עובד עם **תאוצות** ו**עצירות שונות באוויר ועל הקרקע**. כדי לשכפל זאת:

```
הכוח הנגד = MoveSpeed × כיוון
התאוצה אל הכוח = GroundAccel (0.75) בקרקע, AirAccel (0.42) באוויר
העצירה = GroundDecel (0.55) בקרקע, AirDecel (0.16) באוויר
```

כלומר, ישנם **6 קבועים פיזיקליים** שיש לכוונן בקפידה כדי שהמשחק ירגיש נכון.

### מורכבות ההתנגשויות

כל פריים, כל אובייקט נע (שחקן, אויבים, פטריות, מטבעות) עלול להיכנס לתוך פלטפורמה. פתרון "תקוע בתוך קיר" דורש:

1. זיהוי כל 4 כיווני חדירה
2. בחירת הכיוון הקצר ביותר להוצאה
3. טיפול בפינות (corner cases) כמו נחיתה תוך קפיצה בו-זמנית

### מורכבות הבינה המלאכותית

מרחב הפרמטרים של רשת נוירונים עם 4 כניסות, 2 שכבות נסתרות (6 ו-4 נוירונים) ו-2 יציאות = **כ-60 משקולות**. לא ניתן לחפש בצורה ממצה. הפתרון: **אלגוריתם גנטי** שמאמץ את הפתרונות הטובים ומשמיד את הרעים.

## 2.3 מי המשתמשים?

| משתמש | מה הוא רוצה |
|-------|------------|
| **שחקן** | להנות ממשחק פלטפורמה חלק עם אתגר הולך וגדל |
| **חוקר / מסקרן** | לצפות בסוכן הלומד ולשנות פרמטרים (גודל אוכלוסייה, אחוז מוטציה) |
| **מפתח** | לקרוא קוד ברור, מודולרי, עם ארכיטקטורה מוסברת |

## 2.4 הנחות המוצא

1. ה-PC מריץ Windows עם .NET Framework 4.7.2
2. המשחק אינו מרובה משתתפים — שחקן יחיד
3. הבינה המלאכותית לומדת מאפס בכל הפעלה (אין שמירת מצב ב-commit הנוכחי)
4. האנימציה מבוססת sprite-sheets — קבצי PNG חיצוניים בתיקייה `assets/`
